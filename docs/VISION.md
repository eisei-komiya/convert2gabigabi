# VISION.md — convert2gabigabi のビジョン

## このアプリが解決する問題

スマートフォンで画像・動画を変換・圧縮したいとき、既存のアプリには以下の問題がある：

- **意図的に遅い**: 圧縮処理は本来一瞬で終わるはずなのに、広告を見せるためにわざと待たせる設計になっている
- **やることとUIのバランスが悪い**: シンプルな操作なのに、重くて複雑なUIを持たせている
- **機能のミスマッチ**: 画像編集ツールは多機能だが「圧縮」「解像度変更」「拡張子変換」には対応していないことが多い

convert2gabigabi はこれらを解決する。**やることは少ないが、それを速く・シンプルに・正直にやる。**

---

## コアコンセプト：ガビガビ化

アプリ名の由来である「ガビガビ」は、画像を意図的に低解像度・低品質に劣化させる処理を指す。

この「ガビガビ化」は単なるネタ機能ではなく、以下の実用的な用途を持つ：

| 用途 | 説明 |
|---|---|
| 🎨 表現としてのガビガビ | 意図的な低解像度化・ノイズによるアート・ユーモア表現 |
| 📦 ファイルサイズ縮小 | ガビガビ度を上げることで劇的なサイズ削減が可能 |
| 🌐 送信用圧縮 | Discord・SNSなどの容量制限に合わせた調整 |

ガビガビ度（レベル1〜5）の調整により、「ちょっと粗い」から「完全に崩壊した」まで自由に制御できる。

## 提供する機能群

### やること

| 機能 | 説明 |
|---|---|
| ガビガビ化 | 縮小率・ガビガビレベルで画像を劣化させる |
| フォーマット変換 | JPEG / PNG / WebP / HEIC（iOS撮影画像）/ GIF / TIFF / BMP などの拡張子変換 |
| 圧縮 | 品質指定によるファイルサイズ削減 |
| プラットフォーム合わせ圧縮 | Discord(10MB)・X・Instagram など各サービスの上限に自動で収める |
| 解像度変更 | 指定解像度へのリサイズ |
| 動画対応（将来）| 動画のフォーマット変換・圧縮・解像度変更 |

これらをすべて**一つのシンプルなUI**で提供する。操作ステップは最小限に。

> **フォーマット変換の方針**: iOSで撮影したHEIC画像はAndroidやWebで扱えないケースがある。
> HEIC → JPEG/PNG への変換は特に優先度が高い。
> 動画ではMP4 / MOV / WebM などを対象とする（将来）。

### やらないこと

| 機能 | 理由 |
|---|---|
| フィルター・色調補正 | Lightroom / Snapseed など既に品質の高いアプリが存在する |
| 画像の切り抜き・回転・テキスト合成 | 編集ツールとしての競合は目指さない |
| AI生成・超解像 | スコープ外 |
| クラウド同期・アルバム管理 | Googleフォトなどに任せる |

## 設計原則

### 速さを誠実に
処理が速ければ速いままユーザーに返す。広告のために意図的に遅くしない。
ローカル処理を基本とし、ネットワーク遅延を排除する。

### シンプルなUI
「何ができるか」が一目でわかるUI。操作は選択→変換の2ステップが理想。
設定項目は増やしすぎない。よく使うものだけ。

### 正直な品質表示
変換後のファイルサイズ・解像度・品質を処理前後で明示する。
ユーザーが結果を予測できる状態を作る。

## 技術方針

- **ローカル処理ファースト**: FFmpegKit を主エンジンに採用。サーバー不要。
- **Clean Architecture**: UI / UseCase / Data の分離。機能追加しやすい構造を維持。
- **Android ファースト**: まず Android で動くものを作る。iOS は後追い。

## issueを作る際の判断基準

以下に当てはまるissueは **ビジョンに沿っている**：

- 変換・圧縮・解像度変更の精度・速度・UXを改善する
- 対応フォーマットを追加する（HEIC・GIF・動画など）
- プラットフォーム対応の圧縮プリセットを追加する
- 動画サポートに向けた基盤整備
- エラーが起きたときにユーザーが理解できるメッセージを出す
- ビルド・CI・テストの整備

以下に当てはまるissueは **スコープ外として却下**：

- フィルター・色調補正・スタンプなど「編集」系の機能
- クラウド・同期・バックアップ系の機能
- ソーシャル機能（共有ボタンは除く）
